# 监管后台端页面级原型方案 v1.0

## 1. 文档信息

- 文档名称：监管后台端页面级原型方案
- 版本：v1.0
- 适用范围：监管后台端一期 HTML 原型设计
- 关联方案：`03_监管后台端一期设计方案_v1.0.md`

---

## 2. 原型目标

本方案用于把监管后台端一期的四页工作台进一步细化为可执行的页面结构。

原型阶段重点验证：

1. 四页结构是否足够承载一期目标
2. 页面间跳转是否顺畅
3. 信息密度是否可控
4. 是否体现“治理中枢”而非“全功能后台”

---

## 3. 原型范围

一期原型仅包含以下四个一级页面：

1. 治理驾驶舱
2. 地图工作台
3. 对象档案
4. 积分管理

原型中不单独创建以下一级页面：

- 事件中心
- 设备管理
- 城管管理
- 服务商管理
- 犬主管理
- 复杂权限管理

---

## 4. 总体布局建议

监管后台端建议采用统一的 PC 端后台布局：

```text
顶部栏：系统标题 / 当前区域 / 全局搜索 / 管理员入口
左侧导航：四个一级页面入口
主内容区：页面主体内容
右侧抽屉：详情、筛选或快捷说明（按页面需要出现）
```

### 顶部栏建议元素

- 系统名称：犬只治理监管后台
- 当前区域切换：如“成都市 / 锦江区”
- 全局搜索入口：犬只 ID / 主手机号 / 城管编号 / 服务商名称
- 消息或提醒入口：一期可弱化
- 当前管理员信息：头像 / 名称 / 退出

### 左侧导航建议

```text
治理驾驶舱
地图工作台
对象档案
积分管理
```

## 5. 页面方案

## 5.1 治理驾驶舱

### 页面定位

监管端首页，用于总览态势、识别风险、观察闭环与查看治理成效。

### 页面结构

```text
页面标题区
核心指标区
风险提醒区
待关注事项区
趋势区
治理成效区
```

### 5.1.1 页面标题区

建议内容：

- 页面标题：治理驾驶舱
- 时间范围切换：今日 / 近7日 / 近30日
- 区域筛选：全市 / 区县 / 街道
- 数据更新时间提示

### 5.1.2 核心指标区

建议使用 6 到 8 个指标卡。

推荐指标：

```text
在册犬只数
主号绑定率
设备在线率
今日异常事件
待处理事件
今日闭环率
积分参与率
服务核销次数
```

每个指标卡建议包含：

- 指标名称
- 当前值
- 同比或环比趋势
- 点击后的跳转方向

跳转示例：

- 今日异常事件 -> 地图工作台
- 待处理事件 -> 地图工作台
- 在册犬只数 -> 对象档案
- 积分参与率 -> 积分管理

### 5.1.3 风险提醒区

建议放在首屏核心区域。

内容形式：

- 风险提醒列表
- 规则预警摘要
- 重点对象提示

每条提醒建议包含：

- 风险类型
- 风险等级
- 关联区域
- 关联对象
- 发生时间
- 快捷操作：查看地图 / 查看档案

一期表达方式：

- 采用规则预警或轻量分析文案
- 不包装成复杂 AI 推理系统

### 5.1.4 待关注事项区

用于承接“需要马上看一眼”的事项。

建议展示：

- 高风险事件
- 长时间未闭环事项
- 设备离线集中区域
- 积分低参与区域

表现形式建议为列表卡片。

### 5.1.5 趋势区

建议采用 2 到 3 张轻量趋势图。

可选图表：

- 异常事件趋势
- 闭环处理趋势
- 积分参与趋势

目标是辅助判断变化，不做复杂 BI。

### 5.1.6 治理成效区

体现正向治理结果，避免首页只充满报警。

建议展示：

- 异常下降趋势
- 设备在线改善情况
- 积分参与提升情况
- 服务核销活跃情况

### 核心交互

1. 点击指标卡 -> 跳转相关页面
2. 点击风险项 -> 打开地图或对象详情
3. 切换时间 / 区域 -> 全页数据联动刷新

---

## 5.2 地图工作台

### 页面定位

空间监管主工作台，用于看区域态势、点位分布和对象空间关系。

### 页面结构

```text
页面标题区
筛选工具栏
左侧图层面板
中间地图区域
右侧详情抽屉
底部统计条（可选）
```

### 5.2.1 页面标题区

建议内容：

- 页面标题：地图工作台
- 当前区域
- 时间筛选
- 快捷重置按钮

### 5.2.2 筛选工具栏

建议筛选项：

- 对象类型：犬只 / 事件 / 城管辖区 / 服务商
- 风险等级：高 / 中 / 低
- 状态：正常 / 异常 / 待关注
- 时间范围
- 区域范围

### 5.2.3 左侧图层面板

建议用勾选方式控制图层显隐。

一期默认图层：

```text
犬只点位
事件点位
城管任务区域 / 负责辖区
服务商点位
重点区域 / 禁遛区域
```

说明：

- 不放城管实时定位
- 不放轨迹回放入口

### 5.2.4 中间地图区域

地图主区域需要清楚表达不同对象的层次。

建议视觉区分：

- 犬只：常规圆点
- 事件：高亮风险点
- 城管辖区：轮廓区块
- 服务商：固定服务点图标
- 重点区域：半透明面

### 5.2.5 右侧详情抽屉

点击地图点位后打开。

根据对象类型显示不同内容：

犬只点位：

- 犬只名称 / ID
- 状态
- 主手机号摘要
- 最近定位时间
- 快捷入口：查看档案

事件点位：

- 事件名称
- 风险等级
- 发生时间
- 关联犬只
- 快捷入口：查看犬只档案

服务商点位：

- 名称
- 类型
- 合作状态
- 最近核销时间
- 快捷入口：查看服务商档案

### 5.2.6 底部统计条

可选展示当前筛选条件下的汇总数：

- 犬只数
- 事件数
- 高风险数
- 服务点数

### 核心交互

1. 切换图层显示
2. 切换时间和区域筛选
3. 点击点位 -> 右侧抽屉
4. 从抽屉跳转到对象档案

---

## 5.3 对象档案

### 页面定位

承接从驾驶舱、地图和全局搜索进入的对象详情查询。

### 页面结构

```text
页面标题区
搜索与筛选区
对象类型 tab
列表区
详情区
```

建议做成“左侧列表 + 右侧详情”的工作台结构，而不是单纯详情页。

### 5.3.1 页面标题区

建议内容：

- 页面标题：对象档案
- 全局搜索框
- 区域筛选

### 5.3.2 对象类型 tab

```text
犬只
城管
服务商 / 合作商
```

默认进入“犬只”。

### 5.3.3 左侧列表区

根据当前 tab 切换列表内容。

犬只列表项建议显示：

- 名称
- 犬只 ID
- 状态标签
- 所属区域
- 最近更新时间

城管列表项建议显示：

- 姓名
- 编号
- 所属辖区
- 近期任务量摘要

服务商列表项建议显示：

- 名称
- 类型
- 合作状态
- 最近核销时间

### 5.3.4 犬只详情区

建议分模块展示：

```text
基础信息
狗牌 / 设备信息
主手机号与绑定状态
积分摘要
定位状态
核验记录
行为记录
异常记录
```

建议布局：

- 顶部对象头图卡
- 下方信息卡网格
- 底部记录列表

### 5.3.5 城管详情区

建议展示：

```text
基础信息
所属辖区
任务负责范围
近期核验记录
近期任务处理情况
在管区域摘要
```

### 5.3.6 服务商详情区

一期轻量展示：

```text
基础信息
位置
合作状态
积分核销次数
最近核销时间
服务覆盖摘要
```

不做经营分析大盘。

### 核心交互

1. 全局搜索 -> 定位对象
2. 切换 tab -> 切换主体类型
3. 点击列表项 -> 刷新右侧详情
4. 从犬只详情跳转到积分管理

---

## 5.4 积分管理

### 页面定位

监管端的激励治理工作台，用于看规则、账户、流水和效果。

### 页面结构

```text
页面标题区
指标摘要区
积分规则区
积分账户区
积分流水区
积分效果区
```

### 5.4.1 页面标题区

建议内容：

- 页面标题：积分管理
- 时间范围切换
- 区域筛选

### 5.4.2 指标摘要区

建议指标：

```text
积分参与率
活跃账户数
本期发放积分
本期核销积分
服务核销次数
异常账户数
```

### 5.4.3 积分规则区

展示当前积分体系的规则摘要。

建议内容：

- 规则名称
- 触发行为
- 发放积分
- 状态
- 适用范围

一期只展示，不做复杂规则编辑器。

### 5.4.4 积分账户区

账户以主手机号为核心。

建议字段：

- 主手机号
- 关联犬只数
- 当前积分余额
- 最近活跃时间
- 状态

### 5.4.5 积分流水区

建议展示：

- 时间
- 主手机号
- 关联行为
- 积分变化
- 来源类型
- 备注

### 5.4.6 积分效果区

体现积分机制是否真的产生治理价值。

建议展示：

- 区域参与率对比
- 核销活跃趋势
- 正向行为触发趋势
- 低参与区域提示

### 一期明确限制

- 不开放手动调整积分
- 不做复杂权益商城
- 不做经营分析后台替代能力

### 核心交互

1. 从指标跳到账户或流水
2. 点击账户 -> 可跳转对象档案查看关联犬只
3. 切换时间和区域 -> 页面联动刷新

## 6. 页面间跳转关系

### 驾驶舱到其他页面

```text
指标卡 -> 地图工作台 / 对象档案 / 积分管理
风险提醒 -> 地图工作台 / 对象档案
待关注事项 -> 对象档案
```

### 地图到对象档案

```text
犬只点位 -> 犬只档案
事件点位 -> 犬只档案
服务商点位 -> 服务商档案
```

### 对象档案到积分管理

```text
犬只详情中的积分摘要 -> 积分管理
服务商详情中的核销摘要 -> 积分管理
```

### 全局搜索入口

```text
搜索犬只 ID / 主手机号 -> 对象档案
搜索城管编号 -> 对象档案
搜索服务商名称 -> 对象档案
```

## 7. 原型表现建议

### 视觉方向

监管端建议采用“冷静、专业、治理感强”的视觉风格：

- 主色偏深蓝或政务蓝
- 风险色明确但不过度刺眼
- 卡片化布局保持后台可读性
- 图表和地图简洁优先

### 原型保真度建议

一期建议直接做中高保真 HTML 原型，原因是：

- 页面之间存在较多跳转关系
- 地图、列表、详情、指标联动需要真实感
- 后续更容易直接转为前端实现参考

---

## 8. 原型制作顺序建议

建议按以下顺序制作 HTML 原型：

1. 搭统一后台框架
2. 先做治理驾驶舱
3. 再做地图工作台
4. 再做对象档案
5. 最后做积分管理
6. 补齐页面跳转和演示数据

---

## 9. 一句话落地原则

```text
监管后台端一期原型，不追求后台做全，只追求四页做透：
驾驶舱给方向，
地图给空间，
档案给主体，
积分给闭环。
```
