- 文档类型：信息架构确认稿
- 端口编号：02
- 版本：v1.5
- 日期：2026-07-13
- 状态：已确认，作为 PRD v2.1 与 DESIGN v2.1 的上游输入
- 本次更新：移除首屏独立 KPI，图层入口移至右下角，并删除回到主页与全屏按钮

## 2. 页面目标

领导在一个页面内完成：

```text
直接查看全部电子狗牌犬只定位气泡
→ 缩放、平移或按区域筛选地图
→ 叠加服务商、友好乐园和禁入区域
→ 点击犬只或地图对象查看右侧详情
→ 获取 AI 辅助判断
→ 导出当前地图图片或 PDF 决策快照
```

## 3. 单页结构

```text
┌──────────────────────────────────────────────────────────────────────┐
│ [品牌/区域/更新时间/筛选]                      [AI分析] [导出]       │
│                                                                      │
│                                                       [详情收起柄] │
│                                                                      │
│             全部电子狗牌犬只气泡直接显示的大地图                     │
│                                                                      │
│ [图例]               [AI 摘要]                     [缩放/图层] │
└──────────────────────────────────────────────────────────────────────┘
```

所有信息浮层覆盖在地图上方，右侧详情首次进入默认收起，打开时不压缩地图。

## 4. 区域指标

地图首屏不单独展示 KPI 卡。在册犬只、活跃犬只、高风险事件和服务商数量统一进入右侧区域详情，并随当前查看区域更新。

## 5. 筛选

### 5.1 区域

- 街道
- 社区

区域筛选只改变地图可视范围和当前区域数据，不生成区域数量聚合气泡。

### 5.2 定位时效

- 全部
- 活跃定位：0–5 分钟
- 较久未更新：>5–15 分钟
- 定位失效：>15 分钟

### 5.3 图层

- 犬只信息
- 服务商
- 宠物友好乐园
- 犬只禁入区域

图层入口与放大、缩小按钮统一位于地图右下角，点击图层图标后向左展开筛选菜单。右下角不提供回到主页和全屏按钮。

一期不提供定位来源筛选，不提供犬只 ID、狗牌编号或手机号搜索。

## 6. 地图点位模型

### 6.1 犬只气泡

- 地图只显示电子狗牌犬只
- 所有具有定位数据的犬只均显示独立气泡
- 每个气泡代表一只具体犬只
- 气泡统一使用紫色并显示 `⚡`
- 不显示普通狗牌或犬主手机定位
- 不显示多犬共用手机点位

### 6.2 不聚合规则

- 不按街道、社区或其他范围生成数量聚合气泡
- 缩放地图时犬只气泡仍保持独立
- 不隐藏、不抽样、不将多个犬只合并为一个数量点
- 点位重叠时允许轻量错位、当前选中置顶或通过缩放拉开
- 一期原型使用可控演示点位验证全量显示规则

### 6.3 其他图层

| 图层 | 颜色 | 形态 |
|---|---|---|
| 犬只信息 | 紫色 | 独立电子狗牌气泡 |
| 服务商 | 黄色 | 服务商点位 |
| 宠物友好乐园 | 绿色 | 点位或区域面 |
| 犬只禁入区域 | 红色 | 区域面 |

## 7. 右侧详情状态

右侧详情包括：

1. 默认区域总览
2. 街道 / 社区区域摘要
3. 单犬详情
4. 服务商详情
5. 宠物友好乐园详情
6. 犬只禁入区域详情
7. AI 深度分析

不再包含城管辖区详情、多犬共用手机详情和普通狗牌手机定位详情。

## 8. 默认区域总览

- 区域名称与行政层级
- 在册犬只数
- 活跃定位犬只数
- 电子狗牌犬只数
- 服务商数量
- 高风险事件数
- AI 一句话摘要
- 数据更新时间

## 9. 街道 / 社区区域摘要

- 区域名称
- 行政层级
- 在册犬只数
- 活跃定位犬只数
- 电子狗牌犬只数
- 服务商数量
- 高风险事件数
- AI 区域摘要
- 数据更新时间

区域摘要由区域筛选或当前地图范围触发，不由聚合气泡触发。

## 10. 单犬详情

固定八行，同一编号内的字段同行：

1. 犬只头像 + 犬只姓名
2. 犬只 ID + 狗牌编号
3. 年龄 + 性别 + 品种
4. 狗牌类型：电子狗牌
5. 最后上报时间 + 定位状态
6. 犬主脱敏手机号
7. 是否存在高风险行为
8. 数据更新时间

一期高风险行为只包括闯入犬只禁入区域。无风险时显示“暂无高风险行为”。

## 11. 服务商详情

- 服务商名称
- 服务商类型
- 所属街道、所属社区
- 服务状态
- 服务地址摘要
- 1 公里内电子狗牌犬只数量
- 1 公里内活跃定位犬只数量
- AI 服务覆盖摘要
- 数据更新时间

## 12. 宠物友好乐园详情

- 乐园名称
- 所属街道 / 社区
- 区域内电子狗牌犬只数量
- 周边电子狗牌犬只数量
- 活跃定位犬只数量
- 附近服务商数量
- 当前举办的活动列表
- AI 友好空间覆盖摘要
- 数据更新时间

活动列表最小字段：活动名称、活动时间、活动状态。不展示面积数据。

## 13. 犬只禁入区域详情

- 区域名称
- 区域类型：商场 / 学校 / 其他
- 所属街道 / 社区
- 当前区域内电子狗牌犬只数量
- 当前接近区域电子狗牌犬只数量
- AI 风险摘要
- 数据更新时间

不展示面积数据和定位来源构成。

## 14. AI

### 14.1 一句话摘要

首屏默认显示当前地图的一句话 AI 摘要。

### 14.2 深度分析

1. 结论
2. 数据依据
3. 可能原因
4. 决策建议

AI 主要分析：

- 电子狗牌犬只异常聚集或密度变化
- 电子狗牌犬只进入或接近禁入区域
- 犬只分布与服务商、宠物友好乐园覆盖是否匹配

## 15. 明确移除

- 全部城管信息、辖区图层、巡查任务 KPI 和辖区详情
- 街道与社区数量聚合气泡
- 普通狗牌定位
- 犬主手机定位
- 多犬共用手机点位和列表
- 定位来源筛选
- AI 城管力量匹配分析
- 事件点位和任务点位

## 16. 关键交互

| 用户动作 | 地图变化 | 右侧变化 |
|---|---|---|
| 首次进入 | 高新区总览直接显示全部电子狗牌犬只气泡 | 详情收起，AI 摘要显示 |
| 放大 / 缩小 | 点位保持独立，不生成聚合 | 保持当前详情或收起状态 |
| 选择街道 / 社区 | 地图定位至所选区域 | 打开或刷新区域摘要 |
| 点击犬只气泡 | 高亮头像与右上角 `⚡` 徽标组合点位 | 打开单犬详情 |
| 点击服务商 | 高亮黄色点位 | 打开服务商详情 |
| 点击友好 / 禁入区域 | 高亮绿色 / 红色区域 | 打开对应详情 |
| 调整时效 / 图层 | 刷新地图点位和区域面 | 刷新详情与 AI 摘要 |
| 点击 AI 分析 | 地图保持当前状态 | 打开 AI 四段式分析 |

## 17. 后续数据接入项

1. 电子狗牌实际上报频率
2. 全量点位同时渲染的真实数量与性能基线
3. 重叠点位的真实分布与可读性策略
4. 友好乐园和禁入区域真实边界来源
5. 禁入区域闯入事件接口与去重规则

## 18. 验收标准

- 默认地图直接显示全部电子狗牌犬只气泡
- 不出现任何数量聚合气泡
- 不出现普通狗牌或手机定位点
- 不出现城管信息、图层、KPI 或详情
- 地图左上角不显示独立 KPI，区域指标在右侧详情中展示
- 右下角只保留放大、缩小和图层入口
- 单犬详情保持八行同行规则
- 服务商、友好乐园和禁入区域图层与详情可用
- AI 和导出内容只引用当前保留的数据范围
