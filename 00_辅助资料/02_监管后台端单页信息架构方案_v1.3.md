## 2. 页面目标

领导进入页面后，应在一个视野内完成：

```text
看四项核心态势
→ 看高新区犬只空间分布
→ 切换定位来源、时效和治理资源图层
→ 下钻到街道、社区或单犬
→ 在右侧理解当前对象
→ 获取 AI 辅助判断
→ 导出当前决策快照 PDF
```

页面不建设侧边一级导航，不出现对象管理、积分管理、任务处理或事件处理入口。

## 4. 页面区域定义

### 4.1 顶部栏

固定展示：

- 产品名称：犬只治理监管地图
- 当前区域：高新区 / 当前街道 / 当前社区
- 数据更新时间
- `AI 分析` 按钮
- `导出 PDF` 按钮

不放置全局搜索、通知、账号运营入口或多级导航。账号信息如必须展示，只保留轻量头像或身份标识。

### 4.2 KPI 区

地图上方或左上方浮置四张紧凑 KPI：

1. 在册犬只数
2. 高新区活跃犬只数
3. 当前城管巡查任务数
4. 高风险事件数

每张 KPI 只展示：

- 指标名称
- 主数值
- `高新区全局口径` 标识
- 统计口径短说明或更新时间

不展示趋势折线、同比、环比和长说明，避免 KPI 卡扩大为传统驾驶舱。

#### KPI 交互规则

- 四张 KPI 不可点击
- 不承担筛选、跳转或右侧详情切换
- 不设置容易被理解为可点击的悬停抬升、箭头或链接样式
- 地图进入街道或社区后，四张 KPI 仍显示高新区全局数据

KPI 与地图局部数据通过“高新区全局口径”标识明确区分，避免领导将全局指标误读为当前地图区域指标。

### 4.3 筛选区

采用一行紧凑筛选条：

- 区域：街道 / 社区级联选择
- 定位来源：全部 / 电子狗牌 / 手机定位
- 定位时效：全部 / 活跃 / 较久未更新 / 定位失效
- 图层：展开五类图层开关
- 重置：恢复高新区总览和默认图层

筛选后必须同步更新：

- 地图点位和区域
- 聚合气泡数量
- 右侧当前区域摘要
- AI 自动摘要
- PDF 导出的上下文

四项 KPI 始终显示高新区全局数据，不随街道、社区、定位来源、定位时效或图层筛选变化。当前地图区域的数据只在聚合气泡和右侧详情中表达。

### 4.4 地图主体

地图是首屏最大区域，承载：

- 紫色犬只信息
- 蓝色城管辖区
- 黄色服务商
- 绿色宠物友好乐园
- 红色犬只禁入区域

地图状态：

```text
高新区总览
→ 街道聚合
→ 社区聚合
→ 单犬点位 / 多犬共用手机点位
```

地图中保留：

- 面包屑或返回上一级
- 图例
- 放大、缩小、回到默认范围、全屏
- 当前缩放层级提示

地图中不展示：

- 投诉事件点
- 异常事件点
- 待处理任务点
- 单个城管位置
- 轨迹回放

### 4.5 右侧上下文详情区

右侧详情区保持固定位置，随用户选择切换内容。

采用三种一级状态：

1. 默认态：当前区域总览
2. 选中态：地图对象详情
3. AI 态：当前地图上下文的深度分析

右侧区不弹出第二层抽屉，不跳转独立档案页，避免连续遮挡地图。

## 5. 默认态：高新区 / 当前区域总览

未选中具体对象时，右侧展示当前区域总览。

### 字段

- 区域名称与行政层级
- 在册犬只数
- 活跃定位犬只数
- 电子狗牌犬只数
- 手机定位犬只数
- 城管数量
- 当前巡查任务数
- 服务商数量
- 高风险事件数
- AI 一句话态势摘要
- 数据更新时间

“高风险事件数”一期只统计犬只闯入禁入区域形成的当前高风险事件。“当前巡查任务数”只统计未完成且未取消的巡查任务。

### 表达原则

- 先给出一句结论，再展示支持结论的关键数字
- 字段按“犬只 → 定位 → 治理力量 → 服务 → 风险”排序
- 不使用多层嵌套卡片

## 6. 右侧详情一：区域聚合详情

点击街道或社区聚合气泡后展示。

### 字段

- 区域名称
- 行政层级：街道 / 社区
- 在册犬只数
- 活跃定位犬只数
- 电子狗牌犬只数及占比
- 手机定位犬只数及占比
- 城管数量
- 当前巡查任务数
- 服务商数量
- 高风险事件数
- AI 区域摘要
- 数据更新时间

### 允许动作

- 继续下钻
- 返回上一级
- 打开 AI 深度分析
- 导出当前区域 PDF

## 7. 右侧详情二：单犬详情

点击近景中的单犬点位后展示。

### 字段

以下编号同时代表原型中的固定行序。用户列为同一条的数据必须在同一行显示：

1. 犬只头像 + 犬只姓名
2. 犬只 ID + 狗牌编号
3. 年龄 + 性别 + 品种
4. 狗牌类型：电子狗牌 / 普通狗牌
5. 最后上报时间 + 定位状态
6. 犬主脱敏手机号
7. 是否存在高风险行为
8. 数据更新时间

当前高风险行为只包括“闯入犬只禁入区域”。无高风险行为时明确显示“暂无高风险行为”，不展示空白模块。

狗牌类型已经承担定位来源区分，因此详情中不再单独增加“定位来源”字段：

- 电子狗牌：位置来自电子狗牌设备
- 普通狗牌：位置来自犬主手机

### 电子狗牌视觉

- 点位与详情标题显示 `⚡`
- 文案显示“电子狗牌定位”
- 在刷新频率确认前不使用“实时精确位置”承诺

### 普通狗牌视觉

- 点位不显示 `⚡`
- 文案明确显示“犬主手机定位”
- 说明该位置用于辅助判断，不代表犬只实时精确位置

### 不展示

- 犬主姓名
- 完整手机号
- 积分账户与流水
- 所属街道和社区的独立字段行
- 附近友好乐园和服务商
- 编辑、维护、处置按钮

## 8. 右侧详情三：多犬共用手机定位

点击关联多只普通狗牌犬只的手机定位点后展示。

### 顶部摘要

- 脱敏手机号 + 关联犬只数量
- 固定显示格式示例：`138******09（3）`

### 犬只列表

每只犬只列表项只显示：

1. 犬只头像 + 犬只姓名
2. 犬只 ID + 狗牌编号
3. 是否存在高风险行为

### 下钻交互

- 点击列表中的某只犬只后，右侧详情区切换为“单犬详情”
- 不打开新页面，不离开地图单页
- 单犬详情提供返回关联犬只列表的入口
- 列表不得把同一手机坐标拆成多条犬只实时位置

## 9. 右侧详情四：城管辖区详情

点击蓝色辖区面或辖区标签后展示。

### 字段

- 辖区名称
- 覆盖街道 / 社区
- 城管数量
- 当前巡查任务数
- 在册犬只数
- 活跃定位犬只数
- 高风险事件数
- AI 监管力量匹配摘要
- 数据更新时间

### 不展示

- 单个城管姓名
- 单个城管位置
- 人员列表
- 任务分派和催办

## 10. 右侧详情五：服务商详情

点击黄色服务商点位后展示。

### 最小字段

- 服务商名称
- 服务商类型
- 所属街道
- 所属社区
- 服务状态
- 服务地址摘要
- 1 公里内犬只数量
- 1 公里内活跃定位犬只数量
- AI 服务覆盖摘要
- 数据更新时间

服务商类型固定为：

- 商品提供商
- 宠物护理服务商，如洗澡、除毛
- 宠物医院，如诊断、疫苗、体检

服务状态固定为：

- 优质
- 普通
- 风险

一期监管端只展示服务状态，不定义评级标准，不提供评级或状态调整能力。

“周边”距离范围统一定义为 1 公里。相关字段在界面中明确标注为“1 公里内犬只”或等价表述。

### 不展示

- 经营分析
- 积分核销流水
- 商家编辑和合作管理

## 11. 右侧详情六：友好乐园 / 禁入区域详情

### 11.1 宠物友好乐园

字段：

- 乐园名称
- 所属街道 / 社区
- 区域内犬只数量
- 周边犬只数量
- 活跃定位犬只数量
- 附近服务商数量
- 当前举办的活动列表
- AI 友好空间覆盖摘要
- 数据更新时间

当前活动列表采用最小字段：活动名称、活动时间、活动状态。

### 11.2 犬只禁入区域

字段：

- 区域名称
- 区域类型：商场 / 学校 / 其他
- 所属街道 / 社区
- 当前区域内犬只数量
- 当前接近区域犬只数量
- AI 风险摘要
- 数据更新时间

“进入”和“接近”的空间判断口径必须在真实数据接入前确定。一期原型只演示结果和数据来源，不虚构算法精度。

## 12. AI 深度分析状态

点击顶部或右侧的 `AI 分析` 后，右侧详情区切换为 AI 态，不额外覆盖地图。

### 内容结构

#### 结论

用一句话说明当前区域最值得关注的空间态势。

#### 数据依据

列出 2 至 4 条可回指地图和当前筛选的数字或空间关系。

#### 可能原因

明确使用“可能”“建议核查”等辅助判断措辞，不把推断写成事实。

#### 决策建议

提供领导层面的资源关注和后续核查建议，不生成任务、不催办、不触发业务动作。

### 上下文条

AI 内容顶部固定展示：

- 当前区域
- 当前缩放层级
- 定位来源筛选
- 定位时效筛选
- 已开启图层
- 数据时间

## 13. PDF 决策快照

PDF 按当前页面状态生成，建议结构：

1. 标题、当前区域、生成时间
2. 四项 KPI
3. 当前筛选和开启图层
4. 当前地图快照与图例
5. 当前区域或对象摘要
6. AI 结论、数据依据、可能原因和决策建议
7. 数据口径与定位来源说明

PDF 不包含编辑入口、交互控件、账号信息和无关页面元素。

## 14. 关键交互状态表

| 用户动作 | 地图变化 | 右侧变化 |
|---|---|---|
| 首次进入 | 高新区总览、街道聚合 | 高新区总览 + AI 一句话摘要 |
| 点击街道聚合 | 缩放并显示社区聚合 | 街道聚合详情 |
| 点击社区聚合 | 缩放并显示单犬点位 | 社区聚合详情 |
| 点击电子狗牌点 | 高亮紫色 `⚡` 点位 | 单犬电子狗牌详情 |
| 点击普通狗牌点 | 高亮紫色普通点位 | 单犬手机定位详情 |
| 点击多犬手机点 | 高亮带数量的普通点位 | 手机定位摘要 + 关联犬只列表 |
| 点击城管辖区 | 高亮蓝色辖区面 | 辖区力量与犬只分布摘要 |
| 点击服务商 | 高亮黄色点位 | 服务覆盖摘要 |
| 点击友好 / 禁入区域 | 高亮绿色 / 红色区域面 | 区域覆盖或风险摘要 |
| 调整筛选 | 重算并刷新当前地图 | 刷新当前上下文和 AI 摘要 |
| 点击 AI 分析 | 地图保持当前状态 | 切换为四段式 AI 深度分析 |
| 导出 PDF | 地图保持当前状态 | 生成当前决策快照 |

## 15. 后续数据接入项

以下问题不阻塞一期 PRD 和演示原型，但在真实数据接入前必须确认：

1. 友好乐园和禁入区域的真实边界数据来源
2. 电子狗牌与手机定位的实际上报频率
3. 城管巡查任务的数据接口与状态映射
4. 禁入区域闯入事件的数据接口与去重规则

## 16. 本稿验收标准

- 地图保持页面绝对主体
- 四项 KPI、筛选、图层、右侧详情和 AI 形成单页闭环
- 行政层级符合“高新区 → 街道 → 社区”
- 电子狗牌和手机定位同图层且可明确区分
- 六类详情均为只读最小信息，不回到对象管理后台
- 事件与任务不以地图点位形式出现
- PDF 能完整记录当前决策上下文
- 服务商类型、服务状态和活动字段具有明确枚举或最小结构
- 高风险事件与当前巡查任务具有明确一期统计口径
