# 一线巡查端管理系统逻辑闭环

一线巡查端是面向一线管理人员的移动巡查工具，核心能力是通过扫码、手动查询和地图定位快速识别犬只信息，并对异常情况进行查看、记录和处理。

一线巡查端不仅用于完成犬只核验、异常查看和任务处理，也需要保障一线巡查人员的工作效率、责任边界和处理安全。  

本端口通过任务优先级排序、地图导航、扫码核验、自动留痕、处理反馈和个人巡查记录，帮助巡查人员减少无效巡查、降低现场沟通成本、提升处理效率，并为个人工作量和处置过程提供可追溯证明。

目前需要完成以下原型结构:

- 登录页
- 首页
- 扫码页
- 手动查询页
- 犬只详情页
- 地图页
- 消息列表页
- 消息详情页
- 我的页

## 1. 登录页

### 1.1 基础功能

- 微信授权登录

- 展示系统名称

- 一句话的功能使用说明

- 授权失败提示

### 1.2 页面重点

- 城管人员的账号属于个人端, 不同的账号的权限不相同, 比如不同的管辖区域. ( 一期先不做权限管理)

- 如果微信账号未绑定城管队员身份, 应该提示相关管理员 (监管端口 )

- 尽量保持精简干净

### 1.3 存在的状态种类

- 未登录状态

- 微信授权中状态

- 授权(登录)成功状态

- 无权限/ 未绑定身份状态

## 2. 首页

### 2.1 基础功能

- 显示当前登录人的基本信息, e.g.: 队员姓名, 管辖区域

- 扫描快捷入口(顶部右上角), e.g.: 扫一扫, 手动录入识别码, 地图定位

- 今日概览: 
  
  - 今日巡查次数
  
  - 今日核验犬只数量 (新录入数, 检查已录入数)
  
  - 待处理消息数
  
  - 异常事件数

- 待处理消息列表, 展示最新三条

## 3. 扫码页

入口: 首页右上角点击扫一扫图标

### 3.1 基础功能

- 调用微信扫码
- 识别狗牌二维码
- 识别失败提示
- 支持重新扫码
- 支持手动输入犬只 ID
- 扫码 / 手动输入ID码后今日犬只详情页

### 3.2 异常状况

- 二维码无效 / 失效

- 犬只不存在 / 未录入

- 狗牌未绑定

- 网络错误

- 权限不足

## 4. 手动查询页

此页面是当二维码扫描功能失效时做的次要查询入口, 支持在查询框内输入二维码下侧的ID码 (类似于美团单车的二维码 )或者犬只ID (这是独一无二的ID码 ).

查询结果:

- 查询成功进入犬只详情页面

- 输入格式错误弹出格式错误 (做一个前端校验 )

- 未找到时显示相应提示

## 6.巡查地图页 (main)

地图页用于辅助城管队员查看当前辖区或当前位置附近的犬只定位、遛狗轨迹、异常事件和重点区域，并支持快速进入犬只核验与事件处理。

地图页初步包括：

- 当前城管位置及其管辖区域
- 犬只定位点 (考虑不同的定位方式做不同的标记, 如电子狗牌实时定位的在狗狗头像右上侧加入一个⚡标)
- 犬只事件摘要
- 重点区域边界, 如禁遛区域, 学校区域
- 筛选功能

### 6.2 主要功能说明

1. 搜索栏: 支持轻量搜索, 根据狗狗ID/狗牌编号/地点搜索某只狗或某个区域的狗狗活动情况, 支持扫一扫按钮

2. 筛选栏 (高级筛选): 考虑根据:
   
   1. **定位来源** (全部定位 / 电子狗牌实时定位 / 犬主手机轨迹定位)
   
   2. **当前状态** (全部状态 / 正常活动 / 需关注活动 (如在学校或禁遛区域周围高频活动) / 高风险活动 (如直接进入禁遛区活动) / 设备离线 )
   
   3. **事件类型** (全部事件 / 禁入区域周围活动事件 / 禁入区域闯入事件 / 设备异常 / 移动轨迹异常 )
   
   4. **时间** (0-15分钟内 / 近一小时  / 今日) 或者( 早晨遛狗时间段 /晚饭后遛狗时间段 )
   
   5. **区域范围** (100m内 / 500m内 / 重点区域附近 / 我的辖区内)

3. 快捷筛选 (可做可不做 ): 全部 / 异常 / 遛狗中 / 电子狗牌定位 / 重点区域附近

4. 点击地图中的狗狗头像气泡弹出底部半屏卡片显示细节, 举例: 
   

### 6.3 默认显示规则

地图默认展示当前位置附近 1km 内的待处理异常、遛狗中犬只、电子狗牌在线犬只和重点区域。  

默认排序规则：高风险优先、距离近优先、待处理优先。
