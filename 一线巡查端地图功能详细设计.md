# 一线巡查端地图功能 PRD

## 1. 功能背景

一线巡查端是面向城管队员的移动巡查工具，原型方案中地图页用于辅助队员查看当前辖区或当前位置附近的犬只定位、遛狗轨迹、异常事件和重点区域，并支持快速进入犬只核验与事件处理。

一期版本进行功能精简，但地图仍作为一线巡查端的核心能力保留。

一期地图不做完整监管地图，不做轨迹回放、不做重点/禁遛区域边界、不做复杂筛选、不做 AI 异常识别。
一期地图只实现轻量巡查能力，为后续地图巡查、区域管控、异常事件联动打基础。

------

## 2. 功能定位

### 2.1 功能名称

**巡查地图**

### 2.2 功能目标

地图功能用于帮助城管队员在移动端查看当前位置附近的犬只定位点，并区分定位来源，支持点击点位查看犬只摘要信息，进一步进入犬只详情或完成基础核验。

### 2.3 一期核心目标

一句话：

> **在地图上展示当前城管位置、附近有定位数据的犬只点位，并支持点击点位查看犬只摘要和进入详情。**

一期保留以下 6 项能力：

1. 展示当前城管位置及其负责的辖区
2. 展示附近有定位数据的犬只点位
3. 区分定位来源：电子狗牌 / 犬主手机
4. 点击犬只点位，弹出底部犬只摘要卡片
5. 从卡片进入犬只详情
6. 支持从任务详情定位到地图上的任务点

------

## 3. 用户角色

### 3.1 主要用户

一线城管队员 / 巡查人员。

### 3.2 用户诉求

用户进入地图后，需要快速知道：

```text
我现在在哪里？
附近有哪些可定位犬只？
这些位置是电子狗牌上传的，还是犬主手机上传的？
这只狗的基础信息是什么？
我能不能进入详情继续核验？
任务点在哪里？
```

------

## 4. 一期功能范围

## 4.1 本期实现

| 功能         | 说明                                   |
| ------------ | -------------------------------------- |
| 当前人员定位 | 获取并展示当前城管队员位置             |
| 附近犬只点位 | 展示当前位置附近有定位数据的犬只       |
| 定位来源区分 | 区分电子狗牌定位与犬主手机定位         |
| 点位点击     | 点击犬只点位后展示底部摘要卡片         |
| 详情跳转     | 支持从摘要卡片进入犬只详情页           |
| 任务定位     | 支持从任务详情页跳转到地图并定位任务点 |

------

## 4.2 本期不实现

| 功能             | 本期不做原因                                  |
| ---------------- | --------------------------------------------- |
| 重点区域边界     | 二期做区域管控能力                            |
| 禁遛区域边界     | 需要后台区域配置，暂缓                        |
| 轨迹回放         | 数据量和地图展示复杂度较高                    |
| 完整遛狗轨迹线   | 一期只展示当前位置点位                        |
| 地图复杂筛选     | 一期只做最小筛选或不做高级筛选                |
| 热力图           | 更适合监管后台                                |
| AI 异常识别      | 后期数据积累后再做                            |
| 导航前往         | 一期可暂不做，后续接地图 SDK                  |
| 拍照上传         | 非地图一期核心                                |
| 设备原始数据展示 | 不在一线端展示 rssi、accuracy、heading 等字段 |

# 7. 地图展示规则

## 7.1 当前城管位置

### 展示内容

地图加载后，默认尝试获取当前城管队员位置。

成功后：

```text
地图中心定位到当前用户位置
展示当前人员位置标记
加载当前位置附近犬只点位
```

### 定位失败

如果用户未授权定位，展示提示：

```text
需要获取当前位置，才能查看附近犬只定位。
[开启定位权限]
```

如果定位失败：

```text
当前位置获取失败，请检查定位权限或网络状态。
[重新定位]
```

------

## 7.2 犬只点位展示

一期只展示**有定位数据的犬只**。

包括：

1. 电子狗牌上传的最近定位
2. 犬主人端开启“开始遛狗”后上传的手机定位

普通狗牌如果没有犬主手机定位数据，不在地图上展示。

### 点位字段

每个犬只点位至少需要：

```json
{
  "dogId": "D023",
  "dogName": "豆豆",
  "dogBreed": "金毛",
  "tagType": "electronic",
  "locationSource": "device",
  "longitude": 104.123456,
  "latitude": 30.123456,
  "locationTime": "2026-07-04 10:24:00",
  "registerStatus": "registered",
  "currentStatus": "normal"
}
```

------

## 7.3 定位来源区分

地图点位必须区分定位来源。

### 定位来源枚举

| 字段值        | 展示文案 | 含义                        |
| ------------- | -------- | --------------------------- |
| `device`      | 电子狗牌 | 由电子狗牌设备上传          |
| `owner_phone` | 犬主手机 | 犬主人开启遛狗后由手机上传  |
| `task`        | 任务位置 | 由任务 / 异常消息提供的位置 |

### 展示建议

| 类型         | 地图表现                       |
| ------------ | ------------------------------ |
| 电子狗牌定位 | 狗牌图标 / 犬只图标 + 电子标识 |
| 犬主手机定位 | 手机 / 遛狗图标                |
| 任务点       | 任务标记 / 异常标记            |
| 当前城管位置 | 当前定位圆点                   |

不要只依赖颜色区分，最好通过图标区分。户外看手机，颜色经常像被太阳亲自审判过。

# 8. 地图加载逻辑

## 8.1 默认加载范围

地图进入后默认加载当前位置附近犬只。

建议默认范围：

```text
当前位置附近 1km
```

如后端支持，可由接口参数控制：

```json
{
  "longitude": 104.123456,
  "latitude": 30.123456,
  "radius": 1000
}
```

## 8.2 默认展示数据

默认展示：

```text
电子狗牌定位点
犬主手机遛狗定位点
任务点，若从任务详情进入
```

## 8.3 暂不展示数据

一期不展示：

```text
无定位数据的普通狗牌犬只
历史轨迹点
完整轨迹线
区域边界
热力图
设备状态详情
```

# 9. 筛选功能

一期筛选保持极简。

## 9.1 快捷筛选

地图页顶部展示：

```text
全部｜电子狗牌｜遛狗中
```

可选增加：

```text
异常
```

如果开发时间紧，只做前三个。

## 9.2 筛选规则

| 筛选项     | 展示内容                                       |
| ---------- | ---------------------------------------------- |
| 全部       | 展示所有有定位数据的犬只                       |
| 电子狗牌   | 仅展示定位来源为电子狗牌的犬只                 |
| 遛狗中     | 仅展示定位来源为犬主手机，且状态为遛狗中的犬只 |
| 异常，可选 | 展示 currentStatus 为 abnormal 的犬只或任务点  |

## 10.1 点击犬只点位

点击犬只点位后，地图下方弹出底部摘要卡片。

```text

定位来源：犬主手机
遛狗状态：进行中
最近定位：1分钟前

[查看详情] [标记核验]
```

------

## 10.2 底部卡片按钮

### 查看详情

点击后进入犬只详情页。

传参：

```json
{
  "dogId": "D023"
}
```

### 标记核验

点击后执行简单核验操作。

建议一期弹出确认框：

```text
确认已完成该犬只现场核验？
[取消] [确认]
```

确认后调用接口记录核验结果。

成功提示：

```text
已标记核验
```

失败提示：

```text
标记失败，请稍后重试
```

# 16. UI 展示要求

## 16.1 地图点位

点位数量较多时，允许聚合展示。
一期如果开发时间紧，可以先不做复杂聚合，但至少避免一次性加载过大范围。

## 16.2 底部卡片

底部卡片高度建议控制在屏幕高度的 25% 到 35%。

卡片内容只展示业务摘要，不展示设备原始数据。

## 16.3 设备数据展示原则

前端不展示以下字段：

```text
rssi
accuracy
speed
heading
battery
GPS / 北斗 / Wi-Fi / LBS
离线缓存
断网补传
```

这些字段可用于后端判断定位状态，但不直接展示给城管端用户。

# 17. 验收标准

## 17.1 地图基础能力

- 用户进入地图页后，可以看到当前人员位置。
- 系统可以加载附近犬只定位点。
- 电子狗牌定位和犬主手机定位在视觉上可区分。
- 普通狗牌无犬主手机定位时，不展示在地图上。

## 17.2 点位交互

- 点击犬只点位后，可以弹出底部摘要卡片。
- 摘要卡片展示犬只 ID、犬名、犬种、狗牌类型、定位来源、最近定位时间。
- 点击“查看详情”可以进入犬只详情页。
- 点击“标记核验”可以生成核验记录。

## 17.3 任务定位

- 从任务详情点击“查看位置”后，可以进入地图页。
- 地图可以定位到任务发生位置。
- 地图可以展示任务点和任务摘要卡片。

## 17.4 异常状态

- 定位权限未开启时，页面有明确提示。
- 附近无犬只定位时，页面有空状态提示。
- 地图数据加载失败时，支持重新加载。
- 犬只暂无定位时，不展示错误点位。

```text
TaskSummaryCard：负责任务点底部卡片
patrolMap.ts：封装地图点位接口
dogCheck.ts：封装标记核验接口
```

# 20. 总结

一期地图功能不做大而全，只做轻量巡查地图。

核心闭环是：

```text
进入地图
  ↓
查看附近犬只点位
  ↓
区分定位来源
  ↓
点击点位查看摘要
  ↓
进入犬只详情 / 标记核验
```

最终目标：

> **让城管队员能在地图上快速看到附近可定位犬只，并基于点位进入犬只核验与任务处理。**

这版已经够 Codex 开干了。别让它顺手加一套“智能犬只风险预测系统”，AI 有时也会产品经理附体，场面很难收拾。
