## 2. 本端定位与范围

### 2.1 端定位

> 监管后台端 2.0 面向领导使用，通过高新区犬只治理大地图查看全局 KPI、犬只分布、治理资源与空间风险，并使用 AI 辅助分析和导出能力形成决策依据。

### 2.2 使用角色

| 角色 | 可操作内容 |
|---|---|
| 领导 | 查看高新区全局 KPI、缩放地图、筛选数据、切换图层、查看只读详情、使用 AI 辅助分析、导出图片和 PDF |
| 产品/演示人员 | 使用演示数据切换页面状态，验证地图、详情、AI 和导出闭环 |
| 运营人员 | 不在本端操作；对象与积分数据由运营端维护后供监管端只读展示 |

### 2.3 本期范围

| 类型 | 内容 |
|---|---|
| 包含 | 高新区大地图、四项全局 KPI、区域/定位筛选、五类图层、街道与社区聚合、单犬点位、右侧只读详情、AI 摘要与深度分析、图片与 PDF 导出 |
| 不包含 | 对象管理、积分管理、任务处理、事件处置、单个城管定位、轨迹回放、服务商评级、活动管理、移动端适配 |
| 数据限制 | 一期原型使用可信演示数据，不直接录入或宣称接入真实地图业务数据 |
| 上游依赖 | 犬只、狗牌、手机定位、电子狗牌定位、城管辖区、巡查任务、服务商、友好乐园、禁入区域数据 |

## 3. 设计方向

### 3.1 目的

让领导打开页面后先看到尽可能完整的地图，在低操作负担下快速理解犬只分布，再按需展开细节和 AI 判断。

### 3.2 气质

- 专业、克制、可信
- 地图优先、数据清晰
- 明亮而非深色指挥大屏
- 有品牌温度，但不让品牌色压过地图语义色
- 信息密度适中，避免传统后台的大量表格和层层卡片

### 3.3 记忆点

> “一张完整地图上漂浮必要信息”：地图始终铺满视口，KPI、筛选、AI 摘要和详情都作为可控浮层出现，不切割地图画布。

### 3.4 核心约束

- 宽桌面优先，不考虑移动端
- 主要画布为 `1920 × 1080`
- 最小支持宽度为 `1440px`
- 地图铺满整个可用视口
- 右侧详情默认收起，打开后覆盖地图而不压缩地图
- KPI、筛选、AI 摘要和详情全部悬浮在地图上方
- 不设置侧边导航
- 不使用紫色渐变作为通用装饰；紫色只属于犬只图层
- 不创建卡片套卡片的视觉结构

## 4. 公共业务与交互规则

| 规则 | 影响范围 |
|---|---|
| 四张 KPI 始终显示高新区全局数据 | KPI、地图下钻、区域筛选 |
| KPI 不可点击、不触发筛选或详情 | KPI |
| 高新区 → 街道 → 社区 → 单犬逐级下钻 | 地图聚合、面包屑、返回操作 |
| 电子狗牌与普通狗牌属于同一犬只图层 | 图层、图例、点位 |
| 电子狗牌点位显示 `⚡`，普通狗牌不显示 | 点位、犬只详情 |
| 普通狗牌位置来自犬主手机，不等于犬只实时精确位置 | 点位说明、详情、导出 |
| 城管只展示辖区聚合数量，不展示个人位置 | 城管辖区图层、详情 |
| 高风险行为一期只包括闯入犬只禁入区域 | KPI、犬只详情、禁入区域详情、AI |
| 当前巡查任务只统计未完成且未取消的任务 | KPI、区域详情、辖区详情 |
| 手机号必须脱敏，系统不存在犬主姓名字段 | 犬只详情、多犬列表、导出 |
| AI 只做辅助分析，不自动修改数据或触发任务 | AI 摘要、深度分析 |
| 服务商状态由上游提供，监管端与 AI 均不得自行评级 | 服务商详情、AI |
| 地图筛选不影响四张全局 KPI | 筛选、KPI、导出 |

# 犬只治理监管地图

## 5.1 页面目标

地图首屏同时承载犬只分布、四项全局 KPI、筛选、图层和 AI 摘要。领导点击聚合区域或地图对象时，按需打开右侧详情；任何详情打开都不改变地图尺寸。

## 5.2 首屏骨架

```text
┌──────────────────────────────────────────────────────────────────────┐
│ [品牌/区域/更新时间] [区域/来源/时效/图层]      [AI分析] [导出]     │
│                                                                      │
│ [在册犬只] [活跃犬只] [巡查任务] [高风险事件]           [详情收起柄] │
│                                                                      │
│                         高新区全屏地图                               │
│                                                                      │
│ [图例]             [AI 一句话摘要浮窗]            [缩放/定位/全屏] │
└──────────────────────────────────────────────────────────────────────┘

点击地图对象后：右侧详情以覆盖层打开，地图不重排、不缩窄。
```

## 5.3 页面状态

| 状态 | 触发条件 | 页面展示 |
|---|---|---|
| 首次进入 | 页面加载完成 | 高新区总览、街道聚合、四项全局 KPI、筛选、图例、AI 一句话摘要；右侧详情收起 |
| 街道下钻 | 点击街道聚合气泡 | 地图缩放至街道，显示社区聚合；右侧打开街道详情 |
| 社区下钻 | 点击社区聚合气泡 | 地图缩放至社区近景，显示单犬点位；右侧打开社区详情 |
| 对象选中 | 点击犬只、辖区、服务商、友好乐园或禁入区域 | 高亮对象并打开对应右侧详情 |
| AI 深度分析 | 点击 AI 分析 | 右侧详情切换为 AI 四段式分析 |
| 详情收起 | 点击关闭或收起 | 右侧详情隐藏，地图保持当前区域和筛选；AI 摘要浮窗仍保留 |
| 筛选中 | 修改区域、来源、时效或图层 | 地图、聚合数量、详情和 AI 摘要同步刷新；KPI 不变 |
| 加载中 | 页面或局部数据请求中 | 保留地图背景和浮层骨架，内容显示轻量骨架状态 |
| 空数据 | 当前筛选无匹配结果 | 地图无点位，显示“当前条件下暂无数据”及重置筛选入口 |
| 加载失败 | 地图或数据获取失败 | 显示失败原因和重试入口，不使用虚构数据补位 |

## 5.4 页面模块

| 模块名称 | 模块目标 | 主要展示内容 | 包含功能 |
|---|---|---|---|
| 地图主画布 | 提供完整空间态势背景 | 道路、水系、区域边界、五类治理图层 | 缩放、定位、全屏、区域下钻 |
| 顶部工具栏 | 标识产品和当前上下文，并承载高频地图筛选 | 品牌、产品名、当前区域、更新时间、区域/来源/时效/图层筛选 | 筛选、重置、AI 分析、导出 |
| 全局 KPI | 展示高新区核心事实指标 | 在册犬只、活跃犬只、巡查任务、高风险事件 | 仅查看 |
| 顶部筛选区 | 控制当前地图数据范围，嵌入顶部工具栏中部 | 区域、定位来源、定位时效、图层、重置 | 筛选、重置 |
| 图层与图例 | 解释并控制地图对象 | 五类图层颜色、电子狗牌与普通狗牌区别 | 开关图层、查看图例 |
| 聚合与点位 | 表达犬只数量与对象位置 | 街道/社区聚合、单犬、多犬手机点 | 下钻、选中对象 |
| AI 摘要 | 自动说明当前地图态势 | 一句话结论、数据时间 | 打开深度分析 |
| 右侧详情 | 解释当前区域或对象 | 区域、犬只、辖区、服务商、乐园、禁区、AI | 展开、收起、状态切换 |
| 导出 | 固化当前决策上下文 | 当前地图图片、PDF 决策快照 | 导出图片、导出 PDF |

# 6. 视觉系统

## 6.1 颜色 Token

| 角色 | Token | 色值 | 使用范围 |
|---|---|---|---|
| 品牌主色 | Brand Orange | `#FF8F1F` | 品牌标识、主要按钮、当前选中、AI 入口 |
| 品牌深色 | Brand Deep | `#D96E00` | 按钮按下、重点文字 |
| 品牌浅色 | Brand Soft | `#FFF4E8` | 品牌标签、轻提示背景 |
| 犬只图层 | Dog Purple | `#7C3AED` | 犬只聚合、单犬点位、多犬手机点 |
| 城管辖区 | Jurisdiction Blue | `#3B82F6` | 辖区边界、辖区面、城管数量标签 |
| 服务商 | Provider Yellow | `#F59E0B` | 服务商点位和标签 |
| 友好乐园 | Friendly Green | `#22C55E` | 乐园点位、区域面和活动标识 |
| 禁入区域 | Restricted Red | `#EF4444` | 禁入区域面、高风险外环和风险标签 |
| 主文字 | Text Primary | `#1F2937` | 标题、主数值、详情正文 |
| 次文字 | Text Secondary | `#6B7280` | 说明、时间、辅助字段 |
| 弱文字 | Text Muted | `#9CA3AF` | 空状态、禁用和低优先信息 |
| 面板背景 | Panel | `rgba(255,255,255,0.94)` | KPI、筛选、AI、详情浮窗 |
| 面板边框 | Border | `#E5E7EB` | 浮窗和控件细边框 |
| 地图基底 | Map Base | `#EEF2F7` | 演示地图浅色背景 |

### 6.1.1 颜色使用规则

- 品牌橙只用于品牌和操作强调，不替代图层颜色。
- 紫、蓝、黄、绿、红只表达对应地图对象，不扩散到无关组件。
- 禁入风险使用红色外环或标签，不把犬只点位本体从紫色改为红色。
- 定位时效不更换图层主色，通过透明度、外环和文字状态表达。
- 所有颜色必须同时配合图标、标签或边界形态，不能只靠颜色识别。

## 6.2 字体与字号

| 角色 | 字体 | 字号 | 字重 |
|---|---|---:|---|
| 产品标题 | Noto Sans SC / Microsoft YaHei UI | 18px | 600 |
| KPI 主数值 | 同上 | 24px | 600 |
| KPI 标题 | 同上 | 13px | 500 |
| 浮窗标题 | 同上 | 16px | 600 |
| 正文 | 同上 | 14px | 400 |
| 辅助文字 | 同上 | 12px | 400 |
| 地图标签 | 同上 | 12px | 500 |

# 7. 宽桌面布局规则

## 7.1 画布范围

| 项目 | 1920 主画布 | 1440 兼容画布 |
|---|---:|---:|
| 视口 | `1920 × 1080` | 最小宽度 `1440px`，高度自适应 |
| 页面安全边距 | 20px | 16px |
| 右侧详情展开宽度 | 420px | 380px |
| KPI 单卡建议宽度 | 176–188px | 148–160px |
| KPI 高度 | 84px | 80px |
| 浮层间距 | 12px | 10px |

### 7.1.1 不支持范围

- 不进行移动端、平板竖屏和窄屏布局设计。
- 低于 `1440px` 时不继续压缩信息，应显示“请使用宽桌面访问”或保持最小画布横向滚动。
- 浏览器缩放不应作为正式适配方案。

## 7.2 浮层位置

| 浮层 | 默认位置 | 约束 |
|---|---|---|
| 顶部工具栏 | 页面顶部 | 品牌、上下文、筛选、AI 和导出保持单行，不能跨越到右侧详情区域 |
| 四项 KPI | 品牌工具栏下方，横向排列 | 固定四张，不换成两行 |
| 图例 | 左下角 | 默认可见，可折叠为图例按钮 |
| AI 摘要 | 底部中间偏左 | 不遮挡图例、主要聚合气泡和右侧详情 |
| 地图控件 | 右下角 | 详情展开时整体向左让位，不被详情覆盖 |
| 详情收起柄 | 右侧中部 | 默认可见，面积足够点击 |
| 右侧详情 | 右侧 20px/16px 安全边距内 | 覆盖地图，不改变地图宽度 |

# 9. 全局 KPI 模块

## 模块目标

在不占用地图主体的前提下，始终展示高新区四项核心事实指标。

## 展示内容

| KPI | 口径 | 视觉要求 |
|---|---|---|
| 在册犬只数 | 高新区登记犬只总数 | 主数值 + “高新区全局口径” |
| 高新区活跃犬只数 | 最后上报 0–5 分钟 | 主数值 + 更新时间 |
| 当前城管巡查任务数 | 未完成且未取消 | 主数值，使用中性图标 |
| 高风险事件数 | 犬只闯入禁入区域 | 主数值；存在风险时可使用红色小标签，不改变卡片主体色 |

## 模块状态

| 状态 | 页面展示 | 可操作 |
|---|---|---|
| 正常 | 四张 KPI 完整展示 | 无 |
| 加载中 | 数值位置显示骨架 | 无 |
| 数据缺失 | 数值显示 `--`，辅助文字说明暂无数据 | 无 |

## 业务与视觉规则

- KPI 永远不可点击。
- 不出现箭头、链接、悬停抬升和点击手势。
- 不随地图下钻和筛选变化。
- 四张卡横向单排，不拆成两行。
- 卡片不展示趋势图、同比、环比和长说明。

# 12. AI 摘要与深度分析模块

## 模块目标

在不遮挡大地图的前提下，自动给出当前地图的一句话结论，并允许领导按需查看完整分析。

## 12.1 一句话摘要浮窗

| 项目 | 规则 |
|---|---|
| 位置 | 地图底部中间偏左，避开图例和右侧地图控件 |
| 宽度 | 1920 画布建议 520–620px；1440 画布建议 440–520px |
| 高度 | 64–88px，根据两行文字自适应 |
| 内容 | `AI 辅助摘要` 标签、一句话结论、数据时间、`查看分析` 入口 |
| 文本限制 | 最多两行，超出后截断并通过深度分析查看全文 |
| 常驻规则 | 首屏默认显示；右侧详情打开时仍保留，但不得与详情重叠 |

AI 摘要使用品牌橙作为入口强调，不使用紫色渐变或“智能光效”。

## 12.2 深度分析

点击顶部 `AI 分析` 或摘要中的 `查看分析` 后：

1. 右侧详情自动展开
2. 详情切换为 AI 分析状态
3. 地图保持当前区域、缩放、筛选和图层

内容固定为：

| 区块 | 内容要求 |
|---|---|
| 结论 | 一句话指出当前最值得关注的空间态势 |
| 数据依据 | 2–4 条可回指地图的数量、分布或覆盖关系 |
| 可能原因 | 明确使用“可能”“建议核查”等推断措辞 |
| 决策建议 | 提供资源关注与后续核查建议，不触发任务 |

顶部上下文条必须展示：当前区域、缩放层级、定位来源、定位时效、开启图层和数据时间。

## AI 状态

| 状态 | 页面展示 |
|---|---|
| 生成中 | 保留上下文条，分段显示骨架和“正在分析当前地图” |
| 成功 | 展示四段式结果 |
| 依据不足 | 显示“当前数据不足以形成可靠分析”，列出缺失数据 |
| 失败 | 显示重试入口，不保留伪造的旧结论 |

## AI 禁止项

- 不展示未经校准的置信度。
- 不把可能原因写成事实。
- 不自动创建、分配、催办或处理任务。
- 不自行评价服务商为优质、普通或风险。
- 不脱离当前地图上下文输出泛化建议。

# 17. 动效与可用性规则

| 项目 | 规则 |
|---|---|
| 地图下钻 | 240–320ms 平滑缩放和居中，帮助理解层级变化 |
| 详情展开 | 180–220ms 从右侧滑入并淡入 |
| 详情收起 | 160–200ms 向右收起 |
| 点位选中 | 120–160ms 轻量放大和描边变化 |
| 筛选更新 | 旧点位淡出、新点位淡入，避免整页闪烁 |
| 风险提示 | 允许一次性轻脉冲，不允许持续高频闪烁 |
| 减少动态 | 系统开启减少动态时，取消缩放过渡和脉冲，仅保留状态切换 |

### 17.1 键盘与焦点

- 所有按钮、筛选器、图层开关、详情收起柄和列表项必须可获得键盘焦点。
- 焦点样式使用清晰的品牌橙外环，不只依赖颜色变化。
- 打开详情后，焦点进入详情标题；关闭后回到触发对象。
- `Esc` 可收起右侧详情，但不退出浏览器全屏。

### 17.2 对比与信息冗余

- 正文与面板背景对比度满足常规可读要求。
- 图层除颜色外必须使用图标、边界或标签区分。
- 风险状态同时使用红色外环、风险图标和文字。
- 所有地图浮窗在复杂底图上必须保持清晰边界和阴影。

|---|---|

| 单犬同行 | 八行顺序与同行关系严格一致 |
| 手机号 | 全部脱敏，不出现犬主姓名 |
| 多犬列表 | 列表进入单犬详情后可返回 |
| 只读 | 不出现编辑、管理、处置、催办和状态修改 |

## 18.5 AI 与导出

| 验收项 | 标准 |
|---|---|
| AI 摘要 | 首屏默认显示，最多两行 |
| AI 深度分析 | 结论、数据依据、可能原因、决策建议四段完整 |
| AI 边界 | 无置信度、无自动执行、无服务商自主评级 |
| 图片导出 | 包含当前地图、图例、筛选和数据时间 |
| PDF | 只导出当前地图状态决策快照 |
