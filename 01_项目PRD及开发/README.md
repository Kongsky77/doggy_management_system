# 01 项目 PRD 及开发

本目录存放当前有效的产品需求资料，以及依据这些需求制作的原型文件。

设计方案、概念图、交接稿、讨论稿等辅助性资料不应继续堆放在本目录，统一放入 `00_辅助资料/`。

## 文件范围

- 当前有效 PRD、精简方案和功能说明
- 已被采纳并作为执行依据的业务边界说明
- 与当前有效 PRD 对齐的原型说明
- 可直接评审的 HTML 原型

## 管理要求

- PRD 发生变化时，原型应同步调整。
- 如果某份设计稿或讨论稿尚未成为正式需求依据，应移出本目录，放入 `00_辅助资料/`。
- 文件名应包含业务对象、阶段和版本号，例如 `一线巡查端一期PRD_v1.2.md`。
- 当前有效版本应在文档顶部标明版本、日期和状态。
- 已废弃方案优先通过 Git 历史回溯，不在本目录长期堆放多个副本。
