# 监管后台端地图工作台 HTML 原型搭建清单 v1.0

本文档用于把 `1920 x 1024` 地图工作台参考图直接转译为可执行的 HTML 原型搭建规范。它不是 Figma 复刻稿，而是后续编写原型 HTML 时应直接遵循的尺寸、颜色、字体、结构和模块约束。

当前适用对象：
- `01_项目PRD及开发/02_监管后台端一期原型_v1.0.html`
- `01_项目PRD及开发/02_监管后台端DESIGN_v1.0.md`

## 1. 使用原则

1. 不把 Figma 作为 HTML 原型的必经生产环节。
2. 先定页面骨架、模块尺寸和样式 token，再开始改页面。
3. 所有关键尺寸优先遵循 `1920 x 1024` 参考图，减少 AI 自由发挥。
4. 地图工作台应呈现“顶部连续过滤 + 中部大地图 + 右侧详情承接 + 底部统计收口”的工作台结构。

## 14. HTML 结构映射建议

建议地图页结构按以下顺序组织：

1. `map-page-header`
2. `map-filter-bar`
3. `map-layer-bar`
4. `map-main`
5. `map-canvas-card`
6. `map-detail-card`
7. `map-bottom-stats`

推荐 BEM 或近似命名：
- `map-page`
- `map-page__header`
- `map-page__filter-bar`
- `map-page__layer-bar`
- `map-page__main`
- `map-page__canvas`
- `map-page__detail`
- `map-page__stats`

## 15. 搭建顺序建议

1. 先搭左侧导航与右侧主工作区骨架。
2. 再搭标题栏、第一排筛选条、第二排图层开关条。
3. 固定地图区与右侧详情区的宽高关系。
4. 再填地图覆盖物、悬浮控件和图例。
5. 最后补底部统计条。
6. 图标、细节状态和微交互放到最后补。

## 16. 约束说明

后续 AI 或人工继续调整地图页时，应优先遵循本文档，不应随意：
- 改变左导航、地图卡、详情卡、底部统计条的核心宽高关系。
- 把右侧详情压缩成抽屉式窄栏。
- 把底部统计条拆成两行。
- 在顶部额外堆叠解释性文案，破坏工作台密度。

当需要扩展功能时，优先在现有卡片内部做字段替换和轻量增减，而不是先破坏骨架。
