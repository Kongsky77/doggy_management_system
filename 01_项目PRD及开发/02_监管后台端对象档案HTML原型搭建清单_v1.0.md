# 监管后台端对象档案 HTML 原型搭建清单 v1.0

本文档用于把 `1920 x 1024` 的对象档案参考图直接转译为可执行的 HTML 原型搭建规范。它不是 Figma 复刻稿，而是后续编写对象档案页原型 HTML 时应直接遵循的尺寸、颜色、字体、结构和模块约束。

当前适用对象：
- `01_项目PRD及开发/02_监管后台端一期原型_v1.0.html`
- `01_项目PRD及开发/02_监管后台端DESIGN_v1.0.md`

## 1. 使用原则

1. 不把 Figma 作为对象档案页 HTML 原型的必经生产环节。
2. 先固化页面骨架、模块尺寸和样式 token，再开始改页面。
3. 所有关键尺寸优先遵循 `1920 x 1024` 参考图，减少 AI 和人工在实现阶段的自由发挥。
4. 对象档案页应突出“统一查档 + 查看关联 + 快速判断状态”，而不是做成营销页、详情海报或 CRM 风格后台。

## 14. 页面收口规则

- 对象档案页不保留独立底部 KPI 条。
- 当前结果数、高风险犬只、主号未绑定、设备离线、近 7 日已核验、合作服务商、数据更新时间等信息，如后续仍需展示，应优先并入左侧列表头部、右侧对象主卡或业务摘要卡内部，不再单独占据整页底部一整行。
- 页面收口应由“记录区 + 快捷导航区”完成，而不是再加一层统计条。

## 15. 字体规范

- 页面主标题：`20 / 700`

- 列表标题：`14 / 600`

建议：

- 避免过长解释性段落破坏工作台密度。

## 17. HTML 结构映射建议

建议对象档案页结构按以下顺序组织：

1. `archive-page-header`
2. `archive-filter-bar`
3. `archive-type-tabs`
4. `archive-main`
5. `archive-list-panel`
6. `archive-detail-panel`
7. `archive-summary-row`
8. `archive-record-panel`
9. `archive-shortcuts-panel`

推荐 BEM 或近似命名：
- `archive-page`
- `archive-page__header`
- `archive-page__filters`
- `archive-page__tabs`
- `archive-page__main`
- `archive-page__list`
- `archive-page__detail`

## 18. 搭建顺序建议

1. 先固定 `252 + 1668` 的整体骨架。
2. 再搭标题栏、筛选条和对象类型 Tab。
3. 再搭左列表与右详情的双栏。
4. 先做右侧顶部对象主信息卡。
5. 再做状态摘要卡与业务摘要卡。
6. 最后做记录区和快捷导航区。
7. 图标、状态色和按钮细节最后补。

## 19. 约束说明

后续 AI 或人工继续调整对象档案页时，应优先遵循本文档，不应随意：
- 把左侧列表区改成抽屉式或弹层式列表。
- 把右侧详情拆得过碎，失去连续工作台结构。
- 给对象档案页重新加回独立底部 KPI 条。
- 过度压缩对象主卡，导致头像、名称、状态标签不再一眼可识别。

当需要扩展功能时，优先在现有卡片内部做字段替换和轻量增减，而不是先破坏整体骨架。
