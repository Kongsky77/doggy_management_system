# 犬只管理项目工作区

本仓库按“辅助资料 -> 需求定义 -> 决策留痕 -> 能力复用 -> 方法沉淀”的路径管理项目资产。

新会话或新协作者进入项目时，请先阅读根目录 `AGENTS.md`，再阅读本文件和各目录下的 `README.md`。

## 目录结构

```text
犬只管理项目/
├── 00_辅助资料/          设计方案、概念图、讨论稿、交接稿等辅助性资料
├── 01_项目PRD及开发/    当前有效 PRD、功能说明及依据 PRD 制作的原型 HTML
├── 02_设计决策记录/      开发过程中的需求变更、原因、取舍及设计方案
├── 03_skill蒸馏/        从开发过程中提炼的可复用 Skill 与规则
└── 04_最终方法论/        经项目验证后形成的稳定方法论
```

## 工作流

1. `00_辅助资料` 用于提供背景、草案和参考，不直接作为开发执行依据。
2. 当前需求和原型统一维护在 `01_项目PRD及开发`。
3. 需求发生变化时，先在 `02_设计决策记录` 建立决策记录，再同步修改 PRD 和原型。
4. 开发中出现可跨项目复用的能力时，在 `03_skill蒸馏` 提炼为独立 Skill 或规则。
5. 项目验证完成后，将成熟、稳定的实践归纳到 `04_最终方法论`。
6. 每轮有效改动均单独提交并推送 Git，提交说明应准确表达本轮目的。

## 管理原则

- `00_辅助资料` 可以帮助理解业务和方案背景，但不直接替代 PRD、原型和决策记录。
- PRD 是当前需求范围的唯一依据，原型必须与 PRD 保持一致。
- 设计决策记录解释“为什么改变”，不替代 PRD 本身。
- Skill 只收录具备复用价值、输入输出明确、可重复执行的能力。
- 方法论只收录经过项目实践验证的结论，不收录未经验证的设想。
- 历史方案原则上通过 Git 回溯，确需保留的废弃文档应明确标注状态。
