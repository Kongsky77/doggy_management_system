# D023 监管后台端对象档案页按 HTML 清单重构

## 背景

对象档案页已形成 `02_监管后台端对象档案HTML原型搭建清单_v1.0.md`，并通过 D022 确认后续采用“参考图 -> 尺寸与结构清单 -> 直接搭建 HTML 原型”的路径。原页面仍停留在较粗的搜索条、列表和详情卡组合，未完整体现清单要求的筛选条、对象类型 Tab、双栏工作台、状态摘要、业务摘要、记录区和底部统计条。

## 本次变更

- 对象档案页 HTML 调整为筛选条、对象类型 Tab、左列表、右详情和底部统计条的连续结构。
- 左侧对象列表改为固定列表工作区，补充头像、状态、排序和分页区域。
- 右侧详情区改为主对象卡、4 张状态摘要卡、4 张业务摘要卡、记录面板和快捷导航面板。
- 保留犬只、城管队员、服务商 / 合作商三类对象，不新增一期范围外模块。
- 档案详情中的积分归属表达同步为犬只 ID / 狗牌编号方向。

## 决策

对象档案页后续以 `02_监管后台端对象档案HTML原型搭建清单_v1.0.md` 为直接执行依据。页面应优先保持“统一查档 + 查看关联 + 快速判断状态”的工作台结构，不再回到解释性说明面板或零散详情卡堆叠。
