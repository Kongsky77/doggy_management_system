# D050 运营端复用旧监管端原型的通用后台组件

## 1. 背景与触发原因

旧监管后台端一期原型 `02_监管后台端一期原型_v1.0.html` 已因监管端职责收敛而废止，不再作为监管端的需求或实现依据。但其四页工作台中包含已验证的通用后台结构：左侧导航、驾驶舱数据组织、对象档案 Tab 与详情、积分规则与流水表等。

运营后台端一期与这些通用管理能力高度相关。为避免从零重复设计并保持项目体验连续性，决定将旧原型作为运营端的组件复用参考，而不是直接复制其监管职责与页面内容。

## 2. 复用范围

### 2.1 直接复用的结构与交互模式

- 固定左侧导航、顶栏与内容工作区的页面骨架；
- 驾驶舱的指标摘要、趋势区与待关注事项的分层；
- 对象档案的对象 Tab、筛选栏、数据表、右侧详情和页面内跳转；
- 积分规则、积分流水、经营摘要的表格与 Tab 组织；
- 页面切换、侧栏收起、列表选中、详情打开等通用原型交互；
- 基础 Token、状态色、表格行、按钮、标签和空状态的实现结构。

### 2.2 改造后复用

- 将旧“治理驾驶舱”改造为跨端运营脉搏与经营重点；
- 将旧对象档案字段和只读内容改造为运营端可维护的犬只、城管队员、服务商和地图区域；
- 将旧积分只读摘要改造为“对象页只读、积分管理页唯一可调整”的明确边界；
- 将旧积分分析改造为发放、核销、余额和使用率的运营分析；
- 将旧详情区改造为支持编辑、逻辑删除 / 下线、恢复及原因填写的运营抽屉；
- 将旧服务商档案改造为由运营端维护服务商品质、活动关联与有效状态；
- 新增活动管理页面和操作历史能力。

### 2.3 不复用

- 地图工作台及其图层、点位、投诉、风险、事件、任务和城管辖区表达；
- 监管端面向领导的只读数据口径和地图导出能力；
- 旧原型中与监管端职责冲突的城管处置、监管判断和积分治理表述；
- 已被当前监管端 PRD 明确废止的页面范围。

## 3. 实施约束

- 运营端必须新建 `05_` 前缀的 HTML 与独立资产目录，不直接覆盖或修改旧 `02_` 原型；
- 复用的是组件模式、结构和可验证交互，不复用旧业务文案、历史数据和监管职责；
- 所有迁移后的页面仍以 `05_运营后台端一期PRD_v1.0.md` 为需求边界，以 `05_运营后台端DESIGN_v1.0.md` 为视觉与交互依据；
- 若旧组件与当前 PRD 冲突，以当前 PRD 和本决策为准。

## 4. 影响范围

- `01_项目PRD及开发/05_运营后台端DESIGN_v1.0.md`；
- 后续 `05_运营后台端一期原型_v1.0.html` 及独立资产；
- 不影响已废止的旧监管端原型，也不恢复其监管端效力。
