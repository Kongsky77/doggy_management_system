# D037 监管地图减重并固定点位屏幕尺寸

## 1. 背景与触发原因

监管端 2.0 原型视觉评审认为：现有地图底图压迫感较强，道路和区域线条过粗，地图缩放后气泡同步放大，配合光晕后形成明显割裂感，影响领导快速阅读空间关系。

同时，地图下钻后缺少位于右侧、能够直接返回上一级的轻量入口。

## 2. 最终决策

- 地图基底改为低对比浅灰白色，弱化网格、道路、水系和行政区填充。
- 道路、行政边界、城管辖区、友好乐园和禁入区域统一使用细线与低透明度；线宽采用非缩放描边，地图放大时不同比变粗。
- 街道与社区聚合气泡整体缩小，取消所有气泡光晕。
- 聚合气泡、单犬点、多犬手机点和服务商点采用反向缩放补偿，在地图缩放过程中保持稳定的屏幕视觉尺寸。
- 数量差异只通过有限的三级尺寸和文字表达，不使用光晕、脉冲或扩大阴影。
- 进入街道或社区层级后，在地图右侧显示轻量“返回上一级”按钮；街道返回高新区，社区返回所属街道。
- 右侧详情展开时，返回按钮与地图控件共同向左让位，不被详情覆盖。

## 3. 装饰性元素说明

本次所称装饰性元素仅指不承载业务语义的地图颗粒纹理、气泡光晕和过强阴影，不包括图例、AI 摘要、数据时间或“演示地图”数据声明。

Codex 桌面应用右下角显示的桌面宠物不属于监管端 HTML 原型，不在本次修改范围内。

## 4. 未采用方案

### 4.1 仅缩小气泡但继续随地图放大

未采用。进入社区层级后气泡仍会再次变大，无法稳定解决遮挡和压迫问题。

### 4.2 保留光晕并降低透明度

未采用。光晕仍会扩大点位视觉占地，与领导查看型地图的克制方向不一致。

### 4.3 只保留面包屑返回

未采用。面包屑位于顶部且详情展开时可能压缩，右侧轻量按钮更贴近地图下钻操作上下文。
