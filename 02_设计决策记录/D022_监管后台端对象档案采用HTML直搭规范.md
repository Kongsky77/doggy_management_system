# D022_监管后台端对象档案采用 HTML 直搭规范

## 背景

监管后台端对象档案页已有 `1920 x 1024` 的参考图。对象档案页的最终交付物仍然是仓库中的 HTML 原型，而不是 Figma 设计稿。

如果继续走“参考图 -> Figma 复刻 -> HTML 原型”的路径，会增加一层中间生产环节，也容易在二次转译中引入不必要的版式偏移。

## 决策

监管后台端对象档案页后续采用“参考图 -> 尺寸与结构清单 -> 直接搭建 HTML 原型”的执行路径：

1. 不把 Figma 作为对象档案页原型搭建的必经生产环节。
2. 保留并沉淀参考图中的尺寸、字体、颜色、圆角、阴影和布局约束。
3. 先形成面向 HTML 的结构映射表，再进入页面原型改造。
4. 通过数据化的尺寸清单限制 AI 和人工在后续迭代中的过度发散。

## 未采用方案

### 方案 A：完整导入 Figma 后再回写 HTML

未采用。

原因是当前项目的主要产出是 HTML 原型，Figma 在这一阶段不能直接替代最终交付，反而会增加中间损耗。

### 方案 B：仅凭概念图口头描述直接改 HTML

未采用。

原因是缺少明确尺寸、字体和颜色约束时，AI 或人工都容易在实现过程中自行脑补，导致页面逐步偏离参考图。
