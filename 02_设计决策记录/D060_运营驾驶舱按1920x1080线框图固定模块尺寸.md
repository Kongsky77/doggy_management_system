# D060 运营驾驶舱按 1920×1080 线框图固定模块尺寸

## 1. 背景与触发原因

运营端驾驶舱需要在 1920×1080 宽桌面中保持稳定、可扫读的模块比例。新增线框图明确了导航栏、顶栏、KPI、两行分析模块和 AI 分析区的尺寸关系，原型需要严格遵循，并控制模块内部信息量避免溢出。

## 2. 最终布局

```text
左侧导航：255px 宽
顶部栏：93px 高
内容区：左右 25px 安全边距，模块间距 18px

第一行：四 KPI，完整横跨内容区，高 113px
第二行：热力图 / AI 咨询 Top10 / 兑换商品 Top10，比例 2 : 1 : 1，高 362px
第三行：对象与设备 / 积分事项 / 待关注事项 / 服务商与活动，四等分，高 285px
底部：全宽 AI 分析，高 166px
```

## 3. 内容约束

- 四 KPI 只保留端口名称、主数值、变化与一行口径；
- 两个排行榜保留完整 Top10，但压缩为紧凑单行；
- 热力图只保留三档时间切换、区域热度与一行摘要；
- 第三行图表和列表均不得超过固定容器高度，超出时降级为摘要或紧凑展示；
- AI 分析只展示当前运营重点、建议动作和数据依据摘要，不展示个体定位、对话正文或自动执行操作。

## 4. 视觉约束

本次只固定尺寸、位置和内容密度；不改变现有品牌色、语义色、圆角、字体或其他视觉 Token。
