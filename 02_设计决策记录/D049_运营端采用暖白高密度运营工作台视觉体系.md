# D049 运营端采用暖白高密度运营工作台视觉体系

## 1. 背景与触发原因

运营后台端一期已确认包含驾驶舱、对象档案、积分管理和活动管理。其核心工作是高频查看指标、筛选数据、维护对象和追溯操作，不适合沿用监管端“全屏地图 + 悬浮详情”的单页结构。

项目已有 CDHT 橙色品牌方向和柔和结构化后台经验。运营端需要在保持统一品牌识别的同时，建立更适合列表、表单、流水和审计记录的工作台规则。

## 2. 最终决策

运营端一期采用“暖白高密度运营工作台”视觉体系：

- 保留 CDHT 橙色作为品牌与主操作色，避免大面积高饱和橙色背景；
- 使用固定左侧导航、轻量顶栏和内容工作区，四个一级页面保持同一导航骨架；
- 驾驶舱强调跨端数据动向与待关注事项，对象、积分和活动页面强调筛选、表格、详情抽屉与受控编辑；
- 信息按“页面标题与动作 → 筛选 → 摘要 → 主列表 / 主图表”组织，不使用卡片套卡片；
- 详情以右侧抽屉覆盖内容区打开；操作历史以右下角悬浮入口打开侧栏或弹层，不单独占据一级页面；
- 主画布以 `1920 × 1080` 宽桌面为准，最小支持宽度为 `1440px`，不纳入移动端。

## 3. 视觉 Token 继承与约束

- 品牌主色：`#FF8F1F`；
- 品牌深色：`#D96E00`；
- 品牌浅色：`#FFF4E8`；
- 页面背景：暖灰白 `#F7F7F5`；
- 面板背景：`#FFFFFF`；
- 主文字：`#1F2937`；次文字：`#6B7280`；边框：`#E5E7EB`；
- 成功、警示、风险分别使用绿色、琥珀色和红色语义色，不与品牌橙混用；
- 基础间距采用 4px 倍数，面板圆角 12px，表格与输入控件使用低阴影和细边框。

## 4. 未采用方案

### 方案 A：复用监管端全屏地图浮层体系

未采用。运营端以对象维护、积分流水和活动管理为主，浮层地图结构不利于持续筛选、批量查看与表单编辑。

### 方案 B：为每个模块采用独立参考产品风格

未采用。对象、积分与活动会在同一工作日反复切换，必须保持统一 Token、导航和状态表达。

### 方案 C：深色运营指挥大屏

未采用。深色高对比大屏不适合长期阅读表格、表单与流水，也会削弱高频管理操作的可读性。
