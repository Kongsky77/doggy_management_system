# D057 对象档案按当前对象类型新增

## 1. 背景与触发原因

对象档案页的“新增对象”按钮未体现当前对象类型，且在先查看其他对象详情后，新增表单可能沿用详情对象类型，造成操作预期不稳定。

## 2. 最终决策

- 新增按钮随当前对象 Tab 明确显示为“新增犬只 / 城管队员 / 服务商 / 地图区域”；
- 点击按钮只依据当前 Tab 打开对应字段的新增表单，不受此前查看详情影响；
- 四类对象表单分别覆盖 PRD 中的基础必填字段，保存后沿用既有操作留痕提示。

## 3. 影响范围

- `01_项目PRD及开发/05_运营后台端一期原型_v1.0_assets/archive-refinement.js`
- 不改变对象档案的字段规则、删除规则和积分只读边界。
