# D034 监管后台端一期 PRD 重写为单页地图决策产品 v2.0

## 1. 背景

监管端已完成旧四页基线冻结、产品定位重构、地图数据语义、KPI、筛选、右侧详情、AI 和导出范围确认，需要形成新的当前有效 PRD。

## 2. 最终决策

新增：

- `01_项目PRD及开发/02_监管后台端一期PRD_v2.0.md`

并将其设为监管后台端一期当前有效 PRD。

旧文件：

- `01_项目PRD及开发/02_监管后台端一期PRD_v1.1.md`

继续保留为已废止历史方案，不再作为设计、原型和开发依据。

## 3. v2.0 核心变化

- 从四页工作台重构为单页地图决策产品
- 第一用户改为领导
- 第一监管主体改为犬只分布
- 监管端只允许查看、筛选、AI 辅助决策和导出
- 对象与积分管理转移至运营端
- 四项 KPI 固定为高新区全局口径且不可点击
- 地图采用高新区、街道、社区和单犬逐级表达
- 地图承载五类图层，移除事件与任务图层
- 右侧详情收敛为领导扫读信息
- AI 采用自动摘要和四段式深度分析
- 支持当前地图图片和 PDF 决策快照

## 4. 后续执行

后续设计规范和新 HTML 原型必须以 v2.0 为唯一业务依据。

在新设计规范确认前，不直接进入 HTML 视觉搭建。

## 5. 影响范围

- 监管端新设计规范
- 监管端新 HTML 原型及独立资产
- 演示数据
- AI 分析内容
- 图片和 PDF 导出结构
- 后续运营端对旧监管端功能的迁移评估
